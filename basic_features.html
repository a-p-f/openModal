Standard Features

There are a lot of UI/accessibility issues related to modal dialogs/windows. openModal() aims to take care of all of them for you, with zero configuration.

Verify for yourself that:

  • The parent window cannot be scrolled while this is open.
  • Clicking on the parent window does nothing.
  • Pressing the tab key will not move focus to the parent window.
  • Programmatically calling click() or focus() on an element in the parent window has no effect.
  • The iframe has aria-modal="true", which informs assistive technologies that elements in the parent window are not currently interactive. Vimium, for example, generates no "hints" for elements in the parent window.
  • If your modal document sets autofocus on an element, that element will be focused when the modal opens (we've set autofocus on the close button).
  • will focus the autofocus element again
  • The iframe containing the modal window is the last child of the body, and it's z-index is "maxed out", so nothing can ever appear over top of it
